Error creating trace session

Error text

Error creating trace session for API proxy [Proxy Name], revision [Number], environment [Env Name]. 
[Description]

where [Description] is a string that describes the cause of the error.

Example

Error creating trace session for API proxy reviews, revision 3, environment test. 
No message processors are wired with the environment(test) of the organization(streetcarts)

or

Error creating trace session for API proxy reviews, revision 3, environment test. 
Failed to create DebugSession 1486295133165

Cause

This error usually indicates that one of the Message Processors is down, a Message Processor is being restarted, a proxy was incompletely deployed to a Message Processor, or a Message Processor cannot communicate with the Management Server.

Troubleshooting

For Cloud customers, check the Apigee status page for outages

Deployment errors can occur when there is an outage in Edge for the Cloud. Use the following URL to check the current Edge status to see if there is an outage:  https://status.apigee.com

Force undeploy and redeploy of the proxy

In some cases, undeploying and redeploying the proxy revision will correct the problem. Apigee recommends that you do a forced undeploy using the Deployments API with the parameter force=true. If the forced undeploy works, try redeploying the proxy. If the issue is not resolved, contact Apigee Support.

Create a new proxy revision

If undeploy/redeploy does not work, try deploying the proxy as a new revision. Sometimes if you delete certain policies from a proxy and then redeploy as the same revision, you can get a deployment error. See About revisions. If the issue is not resolved, contact Apigee Support. See also What to include when you contact Tech Support or the Community.

Check the deployment status of your proxy

Use the Deployments API to check the deployment status of your proxy to the Message Processors and routers in your environment. This step can indicate the source of the problem. For example, it'll show you which Message Processors or routers have errors.

If necessary, you can include the response information when you contact Apigee Support. See also What to include when you contact Tech Support or the Community.
