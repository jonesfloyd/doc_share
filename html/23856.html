Using SAML authentication

SAML authentication is not supported on the Developer Services portal for Private Cloud at this time.

The Security Assertion Markup Language (SAML) specification defines formats and protocols that enable applications to exchange XML-formatted information for authentication and authorization. Administrators of organizations that have been configured for single sign-on can configure the developer portal to use SAML authentication, as described in the following sections.

Enabling SAML authentication

Contact your Apigee customer support representative to enable SAML authentication on your developer portal.

Viewing the SAML authentication configuration

Once enabled by Apigee, to view the SAML authentication configuration:

  1. Log in to your portal as a user with administrator privileges.
  2. Select Configuration > Dev Portal in the Drupal administration menu.
  3. Click SAML authentication settings to expand the section.
  4. Review the following SAML configuration settings. 
    Configuration Option Description
    SAML bearer-token authentication endpoint URL used to access the SAML bearer token.
    API key Key used to access the SAML bearer token.
    Consumer secret Secret used for accessing a SAML bearer token.
